Fallos y mitigaciones

Riesgos principales del sistema y acciones previstas para reducir impacto.

Microfono no disponible

Impacto: GeriaBot no puede escuchar al usuario.

Mitigacion: mostrar estado claro, pedir permisos del navegador y permitir reintentar desde el boton central.

Reconocimiento de voz impreciso

Impacto: el asistente interpreta mal la frase.

Mitigacion: usar frases cortas, confirmar datos sensibles y mantener visible el estado de escucha.

Respuesta de voz interrumpida

Impacto: el adulto mayor puede cortar una respuesta importante.

Mitigacion: detectar interrupcion, cancelar speech synthesis y volver a escuchar sin bloquear la conversacion.

Backend caido o lento

Impacto: no llegan respuestas de Gemini ni se guarda la ficha remota.

Mitigacion: mostrar error legible, conservar ficha localmente y usar health check para diagnostico.

Gemini no responde

Impacto: el asistente queda sin respuesta inteligente.

Mitigacion: timeout configurado, mensaje de fallo controlado y registro de errores del backend.

Respuesta inadecuada de IA

Impacto: podria entregar contenido poco oportuno para el adulto mayor.

Mitigacion: enviar contexto de perfil, notas del apoderado y evitar usarlo como sustituto medico o de emergencia.

Sesion expirada

Impacto: el apoderado pierde acceso a ficha remota.

Mitigacion: validar token al iniciar, limpiar sesion invalida y solicitar login nuevamente.

Contrasena incorrecta en panel

Impacto: no se puede modificar la ficha del adulto mayor.

Mitigacion: bloquear la ficha detras de reautenticacion y mostrar mensaje sin exponer datos internos.

Google Login no configurado

Impacto: el acceso con Google no aparece o no funciona.

Mitigacion: detectar Client ID vacio, mostrar alternativa con correo y contrasena, y documentar variable de entorno.

Base de datos no disponible

Impacto: no se guardan usuarios, sesiones ni perfiles remotos.

Mitigacion: revisar health check, mantener almacenamiento local temporal y registrar errores de conexion.

Perdida de datos locales

Impacto: se borra la ficha guardada en el navegador.

Mitigacion: sincronizar con backend cuando haya sesion y confirmar antes de usar Limpiar.

Datos sensibles expuestos

Impacto: informacion personal o de salud queda visible a terceros.

Mitigacion: requerir login, proteger panel con contrasena, limitar mensajes de error y no publicar secretos en frontend.

CORS o URL incorrecta

Impacto: frontend y backend no se comunican.

Mitigacion: separar environments local/public/produccion y verificar origen permitido en backend.

Despliegue mal configurado

Impacto: la app funciona localmente pero falla en produccion.

Mitigacion: usar build de produccion, revisar variables de entorno y probar endpoints publicos despues de desplegar.

Navegador incompatible

Impacto: voz, microfono o Google Identity pueden no estar disponibles.

Mitigacion: mostrar mensajes de compatibilidad y priorizar Chrome/Edge actualizados para la experiencia de voz.
